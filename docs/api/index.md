# API Reference

The API reference documentation is generated by [Dokka](https://kotl.in/dokka) from KDoc comments in the source code.

To generate the API docs locally, run:

```bash
./gradlew :dokkaGenerate
```

The output is written to `docs/api/dokka/`. Open [`docs/api/dokka/index.html`](dokka/index.html) in a browser to browse the full API reference.

---

## Modules

| Module | Description |
|--------|-------------|
| `formver.annotations` | User-facing annotations (`@AlwaysVerify`, `@Pure`, `@Unique`, …) and built-in specification functions |
| `formver.common` | Shared configuration types (`PluginConfiguration`, `LogLevel`, `ErrorStyle`, …) |
| `formver.compiler-plugin:core` | FIR-to-Viper conversion engine (embeddings, linearization, name mangling, purity) |
| `formver.compiler-plugin:plugin` | K2 checker integration and error reporting |
| `formver.compiler-plugin:viper` | Viper AST definitions and Silicon bridge |
| `formver.compiler-plugin:uniqueness` | Ownership/uniqueness analysis |
| `formver.compiler-plugin:cli` | Command-line option parsing |
| `formver.gradle-plugin` | Gradle plugin integration |
