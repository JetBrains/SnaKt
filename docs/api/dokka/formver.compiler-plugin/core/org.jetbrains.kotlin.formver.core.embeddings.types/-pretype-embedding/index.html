PretypeEmbedding

interface PretypeEmbedding : RuntimeTypeHolder, TypeInvariantHolder

A representation of a Kotlin type without nullability and uniqueness information.

We explicitly choose not to make this a subtype of TypeEmbedding, even though there is a simple way of treating every PretypeEmbedding as a TypeEmbedding: the goal of the separation into types and pretypes is to avoid one showing up where the other is expected. For example, the naming systems are different, and the equality comparisons would not work.

All pretype embeddings must be data classes or objects!
