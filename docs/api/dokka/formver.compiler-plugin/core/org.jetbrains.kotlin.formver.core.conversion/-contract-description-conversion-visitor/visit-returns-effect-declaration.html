visitReturnsEffectDeclaration

open override fun visitReturnsEffectDeclaration(returnsEffect: KtReturnsEffectDeclaration<ConeKotlinType, ConeDiagnostic>, data: ContractVisitorContext): ExpEmbedding

Converts a returns(value) effect declaration into a Viper expression about the return variable.

Mapping:

  • returns() (wildcard) → true (the effect holds unconditionally on any return).

  • returns(null) → returnVariable == null.

  • returnsNotNull() → returnVariable != null.

  • returns(true) → returnVariable == true.

  • returns(false) → returnVariable == false.

NOTE: the compiler allows contracts like returnsNotNull() on functions with a non-nullable return type without error. In that case, returnsNotNull() maps to true and returns(null) maps to false to avoid a null comparison against a non-nullable value in Viper.
