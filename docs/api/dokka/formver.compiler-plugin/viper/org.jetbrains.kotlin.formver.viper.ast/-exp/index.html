Exp

sealed interface Exp : IntoSilver<Exp>

Sealed hierarchy of Viper expressions.

Each nested class represents one node in the Viper expression language and implements IntoSilver to produce the matching viper.silver.ast node for Silicon.

Every expression carries a type (a Type instance) and the standard Silver AST metadata (pos, info, trafos) that are left at their defaults in normal SnaKt usage.

Kotlin-friendly operator overloads (+, -, *, /, %, !, infix and/or/eq/…) are defined as extension functions at the bottom of the file for ergonomic expression construction.
